DocPipe REST API v1

Base URL: https://api.docpipe.io/v1

All endpoints require an API key via the Authorization: Bearer <key> header.

Authentication

POST /auth/token

Exchange credentials for an access token.

{
  "email": "user@example.com",
  "password": "secret"
}

Response:

{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "expires_in": 3600
}

Documents

POST /documents

Upload a document for processing. Accepts multipart/form-data.

FieldTypeRequiredDescription
filebinaryYesThe document file (PDF, DOCX, PPTX, etc.)
configJSONNoProcessing configuration overrides
callback_urlstringNoWebhook URL for completion notification

GET /documents

List all documents with pagination.

ParameterTypeDefaultDescription
pageinteger1Page number
per_pageinteger20Items per page (max 100)
statusstringallFilter: pending, processing, completed, failed

GET /documents/{id}

Get document details and processing status.

GET /documents/{id}/markdown

Download the converted markdown output.

DELETE /documents/{id}

Delete a document and all associated outputs.

Knowledge Graph

GET /graph/query

Query the knowledge graph built from processed documents.

ParameterTypeRequiredDescription
qstringYesNatural language query
modestringNoSearch mode: naive, local, global, hybrid (default: hybrid)
top_kintegerNoNumber of results (default: 10)
